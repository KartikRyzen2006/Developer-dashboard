DAY1: 
<html>      → Whole webpage
<head>      → Website information
<body>      → Everything visible

<header>    → Top
<nav>       → Navigation menu
<aside>     → Sidebar
<main>      → Main content
<section>   → Group of related content
<article>   → Independent content/card/post
<footer>    → Bottom of page
<div>       → Generic block container
<span>      → Generic inline container
Interview Tip: If someone asks, "Why use semantic tags instead of only <div>?", answer:

Semantic tags make HTML more readable, improve accessibility, help search engines (SEO), and clearly describe the purpose of different parts of a webpage.
